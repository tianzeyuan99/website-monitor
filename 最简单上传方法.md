# 最简单的上传方法（3分钟完成）

## 🎯 方法：使用 GitHub 网页直接上传（无需 Git）

### 步骤1：创建仓库（1分钟）

1. 打开：https://github.com/new
2. 填写：
   - **Repository name**: `website-monitor`（或任意名称）
   - **Description**: `中国海油网站监测工具`
   - 选择 **Public** 或 **Private**
   - ⚠️ **不要勾选** "Add a README file"
   - ⚠️ **不要勾选** "Add .gitignore"
   - ⚠️ **不要勾选** "Choose a license"
3. 点击 **Create repository**

### 步骤2：上传文件（1分钟）

1. 在新建的仓库页面，点击 **"uploading an existing file"** 链接
2. 将整个 `网站监测工具_打包文件` 文件夹中的所有文件**拖拽**到上传区域：
   - ✅ `parse_websites_elements.py`
   - ✅ `website_monitor_app.py`
   - ✅ `requirements.txt`
   - ✅ `templates/index.html`（在templates文件夹中）
   - ✅ `.github/workflows/build-windows.yml`（重要！）
   - ✅ 其他文件（可选）
3. 在页面底部：
   - **Commit message**: `Initial commit`
   - 点击 **Commit changes**

### 步骤3：触发打包（1分钟）

1. 点击仓库顶部的 **Actions** 标签
2. 如果看到黄色提示，点击 **"I understand my workflows, enable them"**
3. 在左侧选择 **"Build Windows Executable"**
4. 点击右侧的 **"Run workflow"** 按钮
5. 选择分支（默认 `main`），再次点击 **"Run workflow"**
6. 等待打包完成（约 5-10 分钟）

### 步骤4：下载 exe

1. 打包完成后，在 **Actions** 页面点击完成的工作流（绿色✓）
2. 滚动到页面底部，找到 **Artifacts** 部分
3. 点击 **"网站监测工具-exe"** 下载 zip 文件
4. 解压后得到 `网站监测工具.exe`

## ❓ 常见问题

**Q: 找不到 Actions 标签？**  
A: 确保 `.github/workflows/build-windows.yml` 文件已上传

**Q: 上传文件时找不到 `.github` 文件夹？**  
A: `.github` 是隐藏文件夹，需要：
- macOS: 按 `Cmd + Shift + .` 显示隐藏文件
- Windows: 在文件资源管理器 → 查看 → 显示隐藏文件

**Q: 打包失败？**  
A: 点击失败的工作流查看错误信息，通常是缺少某个文件
